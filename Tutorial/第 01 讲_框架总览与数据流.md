## 1. 为什么要学习这个框架？

### 1.1 框架的核心价值

LLMBO-MO 是一个**多目标贝叶斯优化框架**，它解决了以下问题：

| 问题 | 传统方法 | LLMBO-MO 方案 |
|------|----------|---------------|
| 多目标权衡 | 需要多次运行 | 一次运行获得 Pareto 前沿 |
| 样本效率低 | 随机/网格搜索浪费 | 贝叶斯优化智能采样 |
| 缺乏物理先验 | 纯数据驱动 | LLM 注入物理知识 |
| 黑盒优化 | 无法解释 | 物理代理函数提供可解释性 |

### 1.2 应用场景

本框架专为**锂电池快充协议优化**设计，但架构可扩展至其他多目标优化问题。

**优化对象**：两阶段恒流 (CC-CC) 充电协议

```
充电过程示意图：

电流 I
  │
I₁├─────────┐
  │         │
  │         │
I₂├─────────┼─────┐
  │         │     │
  └─────────┴─────┴────→ 时间
         SOC₁      SOC_end
  SOC₀
```

**三目标（均最小化）**：
1. 充电时间 `t_charge` [s]
2. 峰值温度 `T_peak` [K]
3. 老化程度 `L_aging` [%]

## 3. 数据流详解

### 3.1 完整优化流程的数据流

```
【初始化阶段】
│
├─► Step 1: 构建所有组件
│   └─► 创建 simulator, database, llm, gp, acquisition
│
├─► Step 2: LLM Touchpoint 1a - 生成耦合矩阵
│   └─► W_time, W_temp, W_aging (3×3 矩阵)
│       └─► 注入 CouplingMatrixManager
│
├─► Step 3: LLM Touchpoint 1b - 生成 warmstart 候选
│   └─► N_ws 个初始候选点 (如 10 个)
│       └─► PyBaMM 评估
│           └─► 结果存入 database
│
└─► Step 4: 采集函数初始化
    └─► μ ← θ_best, σ ← 梯度计算


【主循环阶段】(迭代 T 次)
│
├─► 每次迭代的数据流：
│
│   1. 选权重向量
│      └─► w_vec ← Riesz 集合 [随机选一个]
│
│   2. 更新 Tchebycheff 上下文
│      └─► database.update_tchebycheff_context(w_vec, y_min, y_max, η)
│          └─► 重新计算所有可行解的 f_tch 值
│
│   3. 训练 GP
│      └─► X_train, F_tch ← database.get_train_XY()
│          └─► gp.fit(X_train, F_tch, w_vec, t)
│              └─► 学习长度尺度 l
│              └─► 缓存 Cholesky 分解
│
│   4. LLM Touchpoint 2 - 生成新候选
│      └─► state_dict ← {μ, σ, θ_best, f_min, stagnation, ...}
│          └─► X_candidates ← llm.generate_iteration_candidates(n=15, state_dict)
│
│   5. 采集函数评估
│      └─► α = EI × W_charge
│          └─► 选 top-k (k=3)
│
│   6. PyBaMM 评估
│      └─► for θ in selected_thetas:
│              result = simulator.evaluate(θ)
│              database.add_from_simulator(θ, result)
│
└─► 循环结束


【结果保存】
│
└─► database.save("results/database.json")
```

## 4. 核心公式速览

### 4.1 Tchebycheff 标量化（数据库核心）

```
Eq.1: f_tch = max_i(wᵢ·f̄ᵢ) + η·Σᵢ(wᵢ·f̄ᵢ)

其中：
  - wᵢ: 第 i 个目标的权重 (Σwᵢ=1)
  - f̄ᵢ: 归一化后的目标值
  - η = 0.05: tiebreaker 权重

Eq.2a (log 变换): f̃₃ = log₁₀(aging)

Eq.2b (归一化): f̄ᵢ = (f̃ᵢ - y_min) / (y_max - y_min)
```

**代码对应位置** (`database.py:733-781`):

```python
def _recompute_best(self, update_stagnation: bool = False) -> None:
    # Step 1: 取所有可行解原始目标
    Y_raw = np.array([o.objectives for o in feasible])  # (n, 3)

    # Step 2: log₁₀变换（Eq.2a）
    Y_tilde = Y_raw.copy()
    Y_tilde[:, 2] = np.log10(np.maximum(Y_raw[:, 2], 1e-12))

    # Step 3: 动态 min-max 归一化（Eq.2b）
    denom = self._y_max - self._y_min
    Y_bar = (Y_tilde - self._y_min) / denom

    # Step 4: Tchebycheff + η tiebreaker（Eq.1）
    Wf = w[np.newaxis, :] * Y_bar           # (n, 3)
    F_tch = Wf.max(axis=1) + self._eta * Wf.sum(axis=1)  # (n,)
```

### 4.2 采集函数（acquisition 核心）

```
Eq.14: α(θ) = EI(θ) × W_charge(θ)

Eq.15-16 (EI):
  EI(θ) = (f_min - f̂(θ))·Φ(z) + s(θ)·φ(z)
  z = (f_min - f̂(θ)) / s(θ)

Eq.17 (W_charge):
  W_charge(θ) = Πⱼ N(θⱼ; μⱼ, σⱼ²)
```

### 练习 1-5: 公式推导

给定以下数据：
- 原始目标：Y = [2100s, 312K, 0.0035%]
- 动态边界：y_min = [0.3, 2.49, -2.46], y_max = [0.85, 2.51, -1.97]
- 权重向量：w = [0.4, 0.3, 0.3]
- η = 0.05

**任务**：手动计算 f_tch 值

<details>
<summary>点击查看答案</summary>

```
Step 1: log 变换
  f̃₁ = 2100 (不变)
  f̃₂ = 312 (不变)
  f̃₃ = log₁₀(0.0035) = -2.456

Step 2: 归一化
  f̄₁ = (2100 - 0.3) / (0.85 - 0.3) ≈ 0.545
  f̄₂ = (312 - 2.49) / (2.51 - 2.49) ≈ 0.500
  f̄₃ = (-2.456 - (-2.46)) / (-1.97 - (-2.46)) ≈ 0.008

Step 3: Tchebycheff
  max(w·f̄) = max(0.4×0.545, 0.3×0.500, 0.3×0.008) = max(0.218, 0.150, 0.0024) = 0.218
  Σ(w·f̄) = 0.218 + 0.150 + 0.0024 = 0.3704

  f_tch = 0.218 + 0.05 × 0.3704 = 0.218 + 0.0185 = 0.2365
```

</details>
