# LLMBO-MO: Complete Framework Specification

## 0. Problem Setup

**Decision Variables:**

$$\theta = (I_1,\ SOC_1,\ I_2)$$

| Parameter | Range | Description |
|-----------|-------|-------------|
| I₁ | (0, 8) A open | First-stage CC current |
| SOC₁ | [0.1, 0.7] | Switching SOC |
| I₂ | (0, 8) A open | Second-stage CC current |

Implementation: use [0.01, 7.99] to avoid numerical boundary issues.

Default constraint: I₂ ≤ I₁ (soft suggestion in prompt, not hard-coded — let optimizer discover if exceptions exist).

**Objectives (all minimize):**

$$\min\ f(\theta) = \big(f_1(\theta),\ f_2(\theta),\ f_3(\theta)\big) = \big(t_{charge},\ T_{peak},\ \Delta Q_{aging}\big)$$

**Battery:** LG M50 (NMC811/Graphite), 5Ah, Chen2020 parameter set.

**Charging range:** SOC₀ = 0.1 → SOC_end = 0.8.



### 0.1 Objective Evaluation via PyBaMM

The three objectives are extracted from the PyBaMM SPMe simulation coupled with thermal and SEI degradation submodels:

* **Charging Time ($t_{charge}$):** The total simulation time required for the battery to reach the target $SOC_{end} = 0.8$.
* **Peak Temperature ($T_{peak}$):** The maximum volume-averaged cell temperature observed during the entire charging protocol: $T_{peak} = \max_{t} T(t)$.
* **Aging ($\Delta Q_{aging}$):** Quantified via the irreversible loss of lithium inventory (LLI) driven by Solid Electrolyte Interphase (SEI) growth. Since capacity fade is directly proportional to LLI, it is calculated from the change in SEI thickness: 
    $$\Delta Q_{aging} \propto L_{SEI}(t_{end}) - L_{SEI}(0)$$
    *(Implementation note: The exact formula depends on the chosen PyBaMM SEI submodel, e.g., solvent-diffusion limited or reaction-limited SEI growth).*

## 1. Tchebycheff Multi-Objective Decomposition

At each iteration $t$, randomly select weight vector $\mathbf{w}^{(t)}$ from a predefined set $\mathbf{W}$ (generated by Riesz s-energy method):

$$f^{tch}(\theta) = \max_{i \in \{1,2,3\}} \big\{w_i \cdot \bar{f}_i(\theta)\big\} + \eta \sum_{i=1}^{3} w_i \cdot \bar{f}_i(\theta) \qquad (1)$$

To account for the multi-scale nature of the objectives—specifically the aging metric which can vary by orders of magnitude—a logarithmic transformation is applied to $\Delta Q_{aging}$ before normalization:

$$\tilde{f}_1(\theta) = t_{charge}, \quad \tilde{f}_2(\theta) = T_{peak}, \quad \tilde{f}_3(\theta) = \log_{10}(\Delta Q_{aging}) \qquad (2a)$$

The normalized objectives are then calculated dynamically as new data arrives:

$$\bar{f}_i(\theta) = \frac{\tilde{f}_i(\theta) - \tilde{f}_i^{min}}{\tilde{f}_i^{max} - \tilde{f}_i^{min}} \qquad (2b)$$

* $\eta = 0.05$ (tiebreaker weight to ensure strict Pareto dominance)
* $\tilde{f}^{max}$, $\tilde{f}^{min}$ update dynamically based on the historical bounds in $D_M$.
* Different $\mathbf{w}$ $\rightarrow$ different scalarized subproblems $\rightarrow$ exploring different Pareto regions.

## 2. Physics Proxy Function Ψ(θ)

### 2.1 Derivation from P2D Thermal Model

Starting from the heat equation:

$$\rho C_p \frac{\partial T}{\partial t} = q \qquad (3)$$

where the dominant heat source in the P2D model is:

$$q \approx I^2 \cdot R_{eff}(SOC, T) \qquad (4)$$

Integrating over the full charging process:

$$\Psi(\theta) = \int_0^{t_{end}} I(t)^2 \cdot R_{eff}\big(SOC(t), T(t)\big)\, dt \qquad (5)$$

Under two-stage CC protocol with stage-averaged resistance:

$$\boxed{\Psi(\theta) \approx I_1^2 \cdot \bar{R}_1 \cdot t_1 + I_2^2 \cdot \bar{R}_2 \cdot t_2} \qquad (6)$$

where:

$$t_1 = \frac{(SOC_1 - SOC_0) \cdot Q_{nom}}{I_1}, \quad t_2 = \frac{(SOC_{end} - SOC_1) \cdot Q_{nom}}{I_2} \qquad (7)$$

### 2.2 Gradient of Ψ

$$\frac{\partial \Psi}{\partial I_1} = \bar{R}_1 \cdot (SOC_1 - SOC_0) \cdot Q_{nom} \qquad (8)$$

$$\frac{\partial \Psi}{\partial SOC_1} = I_1 Q_n \left[\frac{\partial \bar{R}_1}{\partial SOC_1}(SOC_1 - SOC_0) + \bar{R}_1\right] + I_2 Q_n \left[\frac{\partial \bar{R}_2}{\partial SOC_1}(SOC_{end} - SOC_1) - \bar{R}_2\right] \qquad (9)$$

$$\frac{\partial \Psi}{\partial I_2} = \bar{R}_2 \cdot (SOC_{end} - SOC_1) \cdot Q_{nom} \qquad (10)$$

Numerical implementation: central finite difference, δ = 1e-5.

## 3. Physics-Informed Composite GP Kernel

### 3.1 Kernel Formulation (Objective-Adaptive)

$$\boxed{k^{(t)}(\theta, \theta') = \exp\!\left(-\frac{\|\tilde{\theta} - \tilde{\theta}'\|^2}{2l^2}\right) + \gamma \cdot \nabla\Psi(\theta)^T \mathbf{W}^{(t)}\, \nabla\Psi(\theta')} \qquad (11)$$

- θ̃ = min-max normalized to [0, 1]
- l = length-scale (learned via MLE)
- γ = coupling strength (initial γ₀ = 0.1)
- **W**^(t) = iteration-dependent 3×3 symmetric PSD coupling matrix

**Adaptive Weight Construction.** The coupling matrix varies across iterations based on the current Tchebycheff weight vector:

$$\boxed{\mathbf{W}^{(t)} = \frac{w_1^{(t)} \mathbf{W}_{time} + w_2^{(t)} \mathbf{W}_{temp} + w_3^{(t)} \mathbf{W}_{aging}}{w_1^{(t)} + w_2^{(t)} + w_3^{(t)}}} \qquad (11')$$

where:
- $\mathbf{W}_{time}, \mathbf{W}_{temp}, \mathbf{W}_{aging}$ are three 3×3 symmetric PSD matrices generated by the LLM at initialization (Touchpoint 1a)
- $(w_1^{(t)}, w_2^{(t)}, w_3^{(t)})$ is the Tchebycheff weight vector sampled at iteration t from the Riesz s-energy set

**Physical Interpretation.** This formulation allows the kernel to adapt its physics-informed structure to the current optimization focus:
- When $w_1^{(t)}$ dominates (emphasizing charging time), $\mathbf{W}^{(t)} \approx \mathbf{W}_{time}$
- When $w_2^{(t)}$ dominates (emphasizing temperature), $\mathbf{W}^{(t)} \approx \mathbf{W}_{temp}$
- When $w_3^{(t)}$ dominates (emphasizing aging), $\mathbf{W}^{(t)} \approx \mathbf{W}_{aging}$

### 3.2 Positive Definiteness

**Theorem.** If $\mathbf{W}_{time}, \mathbf{W}_{temp}, \mathbf{W}_{aging} \succeq 0$, then $k^{(t)}(\theta, \theta')$ is a valid PD kernel for any iteration t.

**Proof.**
1. Each objective-specific matrix is PSD: $\mathbf{W}_{time}, \mathbf{W}_{temp}, \mathbf{W}_{aging} \succeq 0$
2. The convex combination of PSD matrices is PSD: $\mathbf{W}^{(t)} = \sum_i \alpha_i \mathbf{W}_i$ where $\alpha_i = \frac{w_i^{(t)}}{\sum_j w_j^{(t)}} \geq 0$ and $\sum_i \alpha_i = 1$
3. Therefore $\mathbf{W}^{(t)} \succeq 0$
4. By Cholesky decomposition: $\mathbf{W}^{(t)} = \mathbf{L}\mathbf{L}^T$
5. The coupling term becomes: $\nabla\Psi(\theta)^T \mathbf{W}^{(t)} \nabla\Psi(\theta') = \nabla\Psi(\theta)^T \mathbf{L}\mathbf{L}^T \nabla\Psi(\theta') = \phi(\theta)^T \phi(\theta')$ where $\phi(\theta) = \mathbf{L}^T \nabla\Psi(\theta)$
6. This is an inner product kernel, hence PD
7. The sum of PD kernels with positive coefficients is PD. ∎

**PSD Validation.** If any LLM-generated matrix $\mathbf{W}_{time}, \mathbf{W}_{temp}, \mathbf{W}_{aging}$ is not PSD: apply eigenvalue projection $\mathbf{W}_{PSD} = \sum_{\lambda_i > 0} \lambda_i \mathbf{v}_i \mathbf{v}_i^T$.

### 3.3 GP Prediction

$$\hat{f}(\theta) = \hat{\mu} + \mathbf{c}^T \mathbf{C}^{-1}(\mathbf{F}^{tch} - \mathbf{1}\hat{\mu}) \qquad (12)$$

$$s^2(\theta) = \hat{\sigma}^2\!\left(1 - \mathbf{c}^T \mathbf{C}^{-1}\mathbf{c} + \frac{(1 - \mathbf{1}^T \mathbf{C}^{-1}\mathbf{c})^2}{\mathbf{1}^T \mathbf{C}^{-1}\mathbf{1}}\right) \qquad (13)$$

where C_ij = k(θ⁽ⁱ⁾, θ⁽ʲ⁾), c_i = k(θ_new, θ⁽ⁱ⁾) — all computed with the composite kernel.

## 5. Acquisition Function

### 5.1 Scoring Formula

For each LLM-generated candidate θ_cand:

$$\boxed{\alpha(\theta) = EI(\theta) \cdot W_{charge}(\theta)} \qquad (14)$$

### 5.2 Expected Improvement

$$EI(\theta) = \big(f_{min} - \hat{f}(\theta)\big) \cdot \Phi(z) + s(\theta) \cdot \phi(z) \qquad (15)$$

$$z = \frac{f_{min} - \hat{f}(\theta)}{s(\theta)} \qquad (16)$$

- f_min: best f^tch in current dataset
- f̂(θ), s(θ): GP posterior mean and std (from composite kernel)
- Φ(·): standard normal CDF
- φ(·): standard normal PDF

### 5.3 Physics-Based Weighting

$$W_{charge}(\theta) = \prod_{j=1}^{3} \frac{1}{\sqrt{2\pi\sigma_j^2}} \exp\!\left(-\frac{(\theta_j - \mu_j)^2}{2\sigma_j^2}\right) \qquad (17)$$

**Search center μⱼ (dynamic drift):**

$$\mu_j^{(t+1)} = \alpha_t \cdot \mu_j^{(t)} + (1 - \alpha_t) \cdot \theta_j^{best,(t)} \qquad (18)$$

$$\alpha_t = \alpha_{max} \cdot \exp(-t / t_{decay,\alpha}) + \alpha_{min} \qquad (19)$$

| Parameter | Value | Meaning |
|-----------|-------|---------|
| α_max | 0.7 | Initial LLM trust |
| α_min | 0.05 | Minimum LLM trust |
| t_decay,α | 60 | Decay timescale |

Effect: early iterations → μ stays near LLM prior; late iterations → μ tracks θ_best.

**Search range σⱼ (sensitivity-guided):**

$$\sigma_j^{(t)} = \frac{c}{\left|\partial\Psi / \partial\theta_j\right|_{\theta^{best}} + \varepsilon_\sigma} \qquad (20)$$

$$c = \kappa \cdot \max_j \left|\frac{\partial\Psi}{\partial\theta_j}\right|, \quad \kappa \approx 0.20 \qquad (21)$$

ε_σ = 0.001 (prevents division by zero).

Logic: large |∂Ψ/∂θⱼ| → small σⱼ → narrow search (fine-grained); small |∂Ψ/∂θⱼ| → large σⱼ → broad search (coarse).

**Stagnation adaptation:**

$$\sigma_j^{(t+1)} = \sigma_j^{(t)} \cdot \big(1 + \rho \cdot \mathbb{1}[f_{min}^{(t)} = f_{min}^{(t-1)}]\big) \qquad (22)$$

ρ = 0.1. If no improvement, expand search by 10%.

### 5.4 Candidate Selection

Rank 15 candidates by α(θ) descending. Select top N_select = 3 for PyBaMM evaluation.

## 6. Complete Algorithm

```
Algorithm: LLAMBO-MO (Objective-Adaptive Kernel)

Input:  Bounds, t_max, LLM engine, context level ∈ {none, partial, full}
Output: Pareto-optimal charging protocols

─── INITIALIZATION ───
 1  {W_time, W_temp, W_aging} ← LLM.get_coupling_matrices()  // Touchpoint 1a
 2  Validate & project each W to PSD if needed
 3  {θ⁽¹⁾,...,θ⁽¹⁵⁾} ← LLM.warm_start()                      // Touchpoint 1b
 4  Evaluate via PyBaMM (parallel) → D_M
 5  μ ← θ_best from D_M

─── OPTIMIZATION LOOP ───
 6  for t = 0 to t_max − 1 do
 7  │
 8  │  // ── Multi-objective decomposition ──
 9  │  w⁽ᵗ⁾ ← random from W (Riesz s-energy set)
10  │  Scalarize D_M via augmented Tchebycheff (Eq.1) → D_T
11  │
12  │  // ── Adaptive coupling matrix construction ──
13  │  W⁽ᵗ⁾ = (w₁⁽ᵗ⁾·W_time + w₂⁽ᵗ⁾·W_temp + w₃⁽ᵗ⁾·W_aging) / (w₁⁽ᵗ⁾+w₂⁽ᵗ⁾+w₃⁽ᵗ⁾)
14  │
15  │  // ── Physics computation ──
16  │  Compute ∇Ψ at θ_best (Eqs.8-10)
17  │
18  │  // ── GP surrogate (physics-enhanced, objective-adaptive) ──
19  │  Build composite kernel (Eq.11) using W⁽ᵗ⁾, ∇Ψ, γ
20  │  Train GP on D_T → f̂(θ), s(θ)
21  │
22  │  // ── LLM candidate generation ──
23  │  summary ← DatabaseSummarizer(D_M, w⁽ᵗ⁾, ∇Ψ)
24  │  {θ_cand⁽¹⁾,...,θ_cand⁽¹⁵⁾} ← LLM.generate(summary)  // Touchpoint 2
25  │
26  │  // ── Acquisition function scoring ──
27  │  Update μ (Eqs.18-19), σ (Eqs.20-22)
28  │  Score each candidate: α = EI × W_charge (Eqs.14-17)
29  │  Select top N_select = 3 candidates
30  │
31  │  // ── Evaluation & update ──
32  │  Evaluate selected candidates via PyBaMM (parallel)
33  │  D_M ← D_M ∪ {new results}
34  │  Update stagnation counter
35  │  t ← t + 1
36  │
37  end for

─── OUTPUT ───
38  Return non-dominated solutions from D_M
```

## 7. Database Summarizer — How LLM Learns from Data

The LLM cannot see the raw database. Instead, a `DatabaseSummarizer` extracts structured summaries at the requested context level.

### 7.1 Context Level Mapping

| Context Level | What LLM Sees | Token Budget |
|--------------|---------------|-------------|
| **No context** | Problem description + variable bounds only | ~80 tokens |
| **Partial context** | + data_card (best solution, ranges, N) + sensitivity | ~250 tokens |
| **Full context** | + top-5 table + coverage analysis + lessons + Pareto summary + weight | ~600 tokens |

### 7.2 Data Card (Partial Context)

Auto-generated each iteration:

```
<data_card>
Iteration: 25/50
Evaluated: 75 protocols
Best: θ = (3.50, 0.550, 2.20)
  → time=28.3min, temp=45.2°C, aging=0.0156%
Objective ranges:
  time: [18.2, 142.5] min
  temp: [27.8, 68.2] °C
  aging: [0.0012, 0.0891] %
</data_card>

<sensitivity>
|∂Ψ/∂I₁| = 0.141
|∂Ψ/∂SOC₁| = 0.258  (most sensitive)
|∂Ψ/∂I₂| = 0.057
</sensitivity>
```

### 7.3 Full Context Additions

**Top-5 table:** Ranked by current Tchebycheff weight.

**Coverage analysis:** Parameter-space histogram showing data density and blind spots.

**Lessons:** Auto-extracted patterns (temperature violation zones, GP failure regions, improvement trend).

**Pareto summary:** Number of non-dominated solutions, fastest/gentlest extremes, front gaps.

### 7.4 Key Principle

The summarizer is **code-side logic, not LLM memory**. Each prompt is self-contained. No conversation history is passed between iterations. This keeps token cost constant (~2000 tokens/call) and avoids "lost in the middle" attention degradation.

## 8. LLM Memory & Statefulness

### 8.1 Decision: Stateless (No Cross-Iteration Memory)

Each LLM API call is **independent**. The "memory" of past iterations is encoded entirely in the structured summary injected into the prompt.

**Rationale:**
- Token cost stays constant (no accumulation)
- No attention degradation over long conversations
- Enables parallel LLM calls (no shared state)
- Easier to debug and reproduce
- All necessary information is already in the summary

### 8.2 "Pseudo-Memory" via Code-Side State

The Python optimizer maintains all state:

```python
class OptimizerState:
    database: List[Dict]           # all evaluated protocols
    best_f_tch: float              # current best objective
    stagnation_count: int          # consecutive non-improving iters
    mu: np.ndarray                 # search center [3]
    sigma: np.ndarray              # search range [3]
    W_matrix: np.ndarray           # coupling matrix [3×3]
    lessons: List[str]             # accumulated lessons (max 5)
    pareto_front: List[Dict]       # current non-dominated set
```

At each iteration, relevant slices of this state are formatted into the prompt by `DatabaseSummarizer`.

## 9. Parallelism

### 9.1 What Can Be Parallelized

| Operation | Time | Parallel? | Method |
|-----------|------|-----------|--------|
| PyBaMM simulation | 1-10 min | ✅ Yes | `ProcessPoolExecutor` |
| LLM API call | 2-5 sec | ✅ Yes | `asyncio + aiohttp` |
| GP training | 0.1-1 sec | ❌ No | Sequential (needs latest data) |
| AF scoring | <0.1 sec | ❌ No | Trivial, no need |

### 9.2 Simulation Parallelism

N_select = 3 candidates evaluated simultaneously:

```python
with ProcessPoolExecutor(max_workers=3) as pool:
    results = list(pool.map(evaluate_pybamm, selected_thetas))
```

### 9.3 Multi-Weight LLM Parallelism (Advanced)

Sample 2-3 weight vectors simultaneously, call LLM in parallel for each, merge candidates:

```
Parallel LLM calls:
  [LLM(w₁)] → 15 candidates ─┐
  [LLM(w₂)] → 15 candidates ─┼→ 30-45 total → AF scores → top 3
  [LLM(w₃)] → 15 candidates ─┘
```

Wall-clock time: same as single call (~3-5 sec) due to async IO.

Benefit: one iteration explores 2-3 Pareto regions simultaneously.

### 9.4 Per-Iteration Timeline

```
GP train [0.5s] → LLM call [3-5s] → AF score [0.05s] → PyBaMM [parallel, max(sim_times)]
                                                                         ↑
Total ≈ sim_time + 5s overhead                        Dominates total time
```

## 11. Evaluation Metric: Hypervolume (HV)

$$HV(\mathcal{P}, \mathbf{r}) = \text{Vol}\!\left(\bigcup_{k=1}^{K} \big[f(\theta^{(k)}),\, \mathbf{r}\big]\right) \qquad (23)$$

where P is the Pareto front, **r** is the reference point (e.g., worst objective values × 1.1).

- HV captures both solution quality and diversity
- Higher HV = better Pareto front
- Plot HV vs. number of evaluations to compare methods

## 14. LLM Touchpoint Summary

```
                    ┌─────────────────────────┐
                    │   Touchpoint 1a         │
                    │   W_time, W_temp,       │
                    │   W_aging (3 matrices)  │
                    │   (called once)         │
                    └────────────┬────────────┘
                                 ↓
                ┌────────────────┴────────────────┐
                │                                 │
┌──────────────┐│    ┌────────────────┐           │
│ Touchpoint 1b││    │ Iteration t:   │←──────────┘
│ Warm Start   │└───→│ W^(t) = Σ w_i  │
│ (called once)│     │ W_i / Σ w_i    │
└──────┬───────┘     │ (Eq.11')       │
       ↓             └───────┬────────┘
  D_M (initial)              ↓ f̂(θ), s(θ)
       │           ┌───────────────────┐
       │           │  Touchpoint 2     │
       │           │  LLM Candidates   │←── DatabaseSummarizer
       │           │  (every iter)     │    (context level)
       │           └───────┬───────────┘
       │                   ↓ 15 candidates
       │           ┌───────────────────┐
       │           │  Touchpoint 3     │
       │           │  AF = EI × W_chg  │←── μ, σ (from ∇Ψ)
       │           │  Score & Select   │
       │           └───────┬───────────┘
       │                   ↓ top 3
       │           ┌───────────────────┐
       └──────────→│  PyBaMM Eval      │
                   │  (parallel)       │
                   └───────┬───────────┘
                           ↓
                      Update D_M
                      Update μ, σ, α_t
                      Loop back ↑
```

LLM influence across optimization stages:

| Stage | Touchpoint 1a (W_time, W_temp, W_aging) | Touchpoint 2 (candidates) | Touchpoint 3 (σ from ∇Ψ) |
|-------|------------------------------------------|---------------------------|---------------------------|
| Early (t < 0.3·t_max) | Active (fixed matrices stored) | Strong (LLM guides search) | Strong (σ guides AF) |
| Mid (0.3-0.7·t_max) | Combined into W^(t) per iteration | Moderate (data card richer) | Moderate |
| Late (t > 0.7·t_max) | Combined into W^(t) per iteration | Weak (data dominates) | Adapting (σ may expand) |

**Adaptive Kernel Behavior.** The three objective-specific matrices are combined dynamically each iteration:
- When optimizing for time (w₁ large): kernel emphasizes W_time structure
- When optimizing for temperature (w₂ large): kernel emphasizes W_temp structure
- When optimizing for aging (w₃ large): kernel emphasizes W_aging structure

This ensures the physics-informed kernel adapts to the current optimization focus while maintaining permanent physical constraints from initialization.
